# Screenshot Guide for FM - Obsidian File Manager

## Screenshots Needed for Documentation

To complete the community submission, we need the following screenshots:

### 1. Main Interface (screenshot.png)
**Recommended size:** 1200x800px or similar 16:10 ratio

Show:
- File list on the left with several files/folders
- Preview pane on the right showing markdown content
- Current path displayed at top
- A file selected (with the accent color highlight)
- Status bar at bottom with keyboard hints

### 2. Search Mode (search-demo.png) - Optional
Show:

### 3. Context Menu (context-menu.png) - Optional
Show:
- Right-click context menu open on a file
- Available actions visible

## How to Take Screenshots

1. **Install the Plugin in Obsidian:**
   - Copy `main.js`, `manifest.json`, and `styles.css` to:
     `<your-vault>/.obsidian/plugins/fm-file-manager/`
   - Enable the plugin in Obsidian Settings

2. **Open FM:**
   - Press `-` (hyphen key) to open FM
   - Navigate to a folder with good visual examples

3. **Capture Screenshots:**
   - Use your OS screenshot tool:
     - macOS: Cmd+Shift+4, then Space to capture window
     - Windows: Win+Shift+S for snipping tool
     - Linux: Use screenshot tool of your choice
   - Save as `screenshot.png` in repository root

4. **Update README:**
   - Ensure screenshot path in README.md is correct
   - Preview README to verify image displays properly

## Screenshot Checklist

## Tips for Good Screenshots

- Use a clean, organized vault for demonstration
- Select an interesting file with markdown content for preview
- Make sure Obsidian theme is clean (light or dark, whichever looks best)
- Ensure no personal/sensitive information visible
- Capture at a reasonable resolution (not too small, not massive)
